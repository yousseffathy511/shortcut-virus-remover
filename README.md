# Shortcut Virus Remover

> One-click Windows tool that quarantines the **USB shortcut virus / Trojan:*/Runner** family from infected PCs and cleans every connected flash drive.

[![Latest Release](https://img.shields.io/github/v/release/yousseffathy511/shortcut-virus-remover)](https://github.com/yousseffathy511/shortcut-virus-remover/releases/latest)
[![OpenSSF Scorecard](https://api.securityscorecards.dev/projects/github.com/yousseffathy511/shortcut-virus-remover/badge)](https://api.securityscorecards.dev/projects/github.com/yousseffathy511/shortcut-virus-remover)

Developed by **Youssef Fathy** under **Widescreen** company. Open source under the MIT License.

This is the malware Microsoft Defender labels as:

- `Trojan:PowerShell/Runner.PGRA!MTB`
- `Trojan:BAT/Runner.PGRD!MTB`
- and similar `Runner.*` variants

If your USB drive's contents have been replaced by a single
**`<DriveName>.lnk`** shortcut and a hidden `sysvolume` folder, you have
this infection. The folder is not gone - it is just hidden, and the
shortcut is bait.

> **Do not double-click the shortcut.** Doing so launches the malware
> with administrator rights and infects the host PC.

## Verify before you run

Every release attaches an **EXE**, an **EXE.sha256** checksum file, and a
**build-provenance attestation** generated by GitHub. The release notes
also link to the **VirusTotal** scan of that exact build (once the
maintainer has set the `VT_API_KEY` repo secret - see release notes for
status).

1. Compare the SHA-256 of `ShortcutVirusRemover.exe` with the value
   printed in the release notes:

   ```powershell
   Get-FileHash .\ShortcutVirusRemover.exe -Algorithm SHA256
   ```

2. Cross-check the same file on
   [VirusTotal](https://www.virustotal.com/gui/home/upload). The release
   notes contain the direct VirusTotal permalink for that build (or a
   note that automatic uploads are pending).

3. Verify the build-provenance attestation that GitHub generated for the
   EXE. This proves the binary came from this repository's CI workflow:

   ```powershell
   gh attestation verify .\ShortcutVirusRemover.exe `
       --repo yousseffathy511/shortcut-virus-remover
   ```

   You need GitHub CLI 2.49+ and to be signed in (`gh auth login`).

If any of those checks fail, **stop** and open a security advisory at
<https://github.com/yousseffathy511/shortcut-virus-remover/security/advisories/new>.

## Run from source (recommended for cautious users)

This is the most transparent way to run the tool: no EXE, no SmartScreen,
no compiled binary - just the same PowerShell scripts you can read in
this repo.

1. **Unplug** any other USB drives so they cannot be re-infected mid-run.
2. Open **Windows PowerShell as Administrator**.
3. Allow the scripts in this session and dry-run first:

   ```powershell
   Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass -Force
   .\Remove-ShortcutVirus.ps1 -WhatIf -Harden
   ```

   `-WhatIf` reports every action without changing anything.

4. When the dry run looks correct, do the real run:

   ```powershell
   .\Remove-ShortcutVirus.ps1 -Harden
   ```

5. **Reboot** if the script reported any DLL was scheduled for
   delete-on-reboot.
6. Plug the USB back in, then recover your files (no admin needed):

   ```powershell
   .\Recover-UsbFiles.ps1 -Drive E:
   ```

7. **Format the USB drive** before using it again, then run a
   **Microsoft Defender Offline Scan**:
   *Windows Security -> Virus & threat protection -> Scan options ->
   Microsoft Defender Antivirus (offline scan)*.

## One-click installer

For non-technical users, a pre-built executable is published on the
[Releases page](https://github.com/yousseffathy511/shortcut-virus-remover/releases/latest).

1. Download `ShortcutVirusRemover.exe`.
2. **Right-click the file -> Properties -> tick `Unblock` -> OK**.
   Windows tags every file downloaded from the internet, and unblocking
   tells Windows you trust this specific copy.
3. Double-click the executable. Click **Yes** on the Windows UAC prompt.
4. If the tool finds anything to remove, you will see a single
   "Continue? [Y/n]" prompt with a 10 second auto-Yes timer. Press
   `Enter` to accept, or `n` then `Enter` to abort.
5. Wait for the cleanup to finish. The window stays open until you press
   a key.

The cleanup log is at
`%ProgramData%\ShortcutVirusRemover\Logs\shortcut-virus-remover-<timestamp>.log`.
Anything quarantined from `C:\Windows\System32` ends up in
`%ProgramData%\ShortcutVirusRemover\Quarantine\<timestamp>\` with a
`quarantine-manifest.json` listing the original path, SHA-256, and
attributes of each item.

## What this tool does NOT do

This is an explicit list of negatives. Every claim is checkable in the
source. If the source ever drifts from this list, please file an issue.

- **No network calls.** The tool never makes outbound HTTP requests.
  There is no telemetry, no auto-update, no phone-home.
- **No scheduled tasks** are created.
- **No user files are touched** outside of removable drive roots.
- **No broad recursive deletion of `C:\`.**
- **System32 DLL deletion is gated** by the IOC `^u\d{4,}\.dll$`. The
  tool quarantines (moves to `%ProgramData%\ShortcutVirusRemover\Quarantine`)
  before scheduling deletion, so a false positive is recoverable.
- **Service deletion is gated** by the IOC `^u\d{4,}$`.
- **Defender exclusion cleanup is scoped** to:
  * `C:\Windows\System32` (with and without trailing backslash)
  * The root of every removable drive (`E:\`, `F:\`, ...)
  * `<RemovableDriveRoot>\sysvolume`

  The tool never touches an exclusion the user added on purpose.
- **USB cleanup deletes only**:
  * Files at the drive root whose extension is in
    `.lnk .vbs .vbe .bat .cmd .js .jse .wsf .scr .exe .hta`
  * Folders at the drive root whose name is `sysvolume`,
    `sysvolume.x86`, or `systemvolume`
  * Files anywhere on the drive matching
    `^u\d{4,}\.(vbs|bat|cmd|js|jse|wsf|dat|bin)$`
  * `.lnk` files in nested folders
- **Restore checkpoint is best effort.** Windows throttles to one
  restore point per 24h by default; the tool logs and continues if a new
  one cannot be created.

## How the malware works

```
USB inserted
    -> User sees only a single ".lnk" file with the volume's name.
    -> Real folder is still there, but hidden + system.

User clicks the .lnk
    -> Runs:  wscript.exe   E:\sysvolume\u123456.vbs
    -> .vbs re-runs itself with "runas" (UAC prompt).
    -> Once elevated, .vbs runs:
            cmd.exe /c "E:\sysvolume\u654321.bat"

The .bat then:
    1. Add-MpPreference -ExclusionPath  E:\
    2. Add-MpPreference -ExclusionPath  C:\Windows\System32
    3. copy  E:\sysvolume\u<n>.dat  ->  C:\Windows\System32\u<n>.dll
    4. rundll32.exe  C:\Windows\System32\u<n>.dll,IdllEntry 1

Persistence on the host PC:
    HKLM\SYSTEM\CurrentControlSet\Services\u<n>
        ImagePath = svchost.exe -k DcomLaunch
        Parameters\ServiceDll = C:\Windows\System32\u<n>.dll

The service watches for new USB drives and re-creates the
shortcut + sysvolume payload on every drive that is plugged in.
```

This is why **deleting the shortcut by hand keeps "failing"**: the
service on the host immediately re-creates it.

### Indicators of compromise

| Where                                    | What to look for                                    |
|------------------------------------------|-----------------------------------------------------|
| `C:\Windows\System32\`                   | A DLL named `u<digits>.dll` (e.g. `u707321.dll`)    |
| `HKLM\SYSTEM\CurrentControlSet\Services` | A service named `u<digits>`                         |
| Defender exclusions                      | `E:\`, `E:\sysvolume`, `C:\Windows\System32`        |
| USB root                                 | `<VolumeLabel>.lnk` shortcut + hidden `sysvolume\`  |
| USB `sysvolume`                          | Files matching `u\d+\.(vbs|bat|dat|bin)`            |
| `rundll32.exe` command lines             | `...,IdllEntry 1`                                   |

## FAQ

**Why does Defender keep detecting `Trojan:PowerShell/Runner.PGRA!MTB`?**
The malware persists as a Windows service that re-creates its dropper
files on every USB insert. Until you stop and delete the `u<digits>`
service and quarantine the System32 DLL it loads, Defender will keep
catching the dropper. **This tool handles that automatically.**

**Why does my USB only show one `.lnk` file?**
The malware sets `+h +s` on your real top-level folder so File Explorer
hides it, then drops a fake shortcut with the same name as the volume
label. Your files are not lost. **This tool handles that automatically**
by clearing `+h +s` on the real folder and deleting the fake shortcut.

**What is the `sysvolume` folder?**
It is a hidden folder dropped at the root of the USB by the malware. It
contains `u<digits>.vbs`, `u<digits>.bat`, and `u<digits>.dat` (the DLL
payload). It is not created by Windows. **This tool deletes it
automatically.**

**Is it safe to delete `u123456.dll`?**
Only if the file lives under `C:\Windows\System32` and matches the IOC
`^u\d{4,}\.dll$`. To be cautious, this tool does **not** straight-delete
that DLL: it moves it into
`%ProgramData%\ShortcutVirusRemover\Quarantine\<timestamp>\` and writes
a `quarantine-manifest.json` so a false positive can be recovered. If
the file is locked by the malware service, the tool falls back to
`MoveFileEx(MOVEFILE_DELAY_UNTIL_REBOOT)` after renaming it.

**Why does SmartScreen warn about the `.exe`?**
Because the binary is not yet code-signed. SmartScreen will warn the
first time any free, unsigned tool is run. You can either run from
source (see "Run from source" above) or right-click the EXE -> Properties
-> Unblock before double-clicking it. Code signing through SignPath
Foundation / Microsoft Trusted Signing is on the roadmap.

**Will this delete my real files?**
No. The tool's USB cleanup only deletes files matching the malware
naming patterns above plus `.lnk` shortcuts. Everything else is left
untouched. Use `Recover-UsbFiles.ps1` to copy your real files to a clean
folder before formatting the USB.

**Does this work on the latest Windows 11?**
Yes. The script uses Windows PowerShell 5.1 features that ship with
Windows 10 and Windows 11. The EXE is built on `windows-latest` runners
in GitHub Actions and tested on the same.

**How do I recover something the tool quarantined?**
Open the quarantine folder printed in the cleanup summary
(`%ProgramData%\ShortcutVirusRemover\Quarantine\<timestamp>\`). The
`quarantine-manifest.json` records the original path. To restore an
item, copy it back to the original path **as Administrator**. If you
believe a quarantined file was a false positive, please file an issue
with the manifest entry attached so the IOC pattern can be tightened.

## Advanced usage

```powershell
# Full disinfection with hardening (the recommended invocation).
.\Remove-ShortcutVirus.ps1 -Harden

# Dry run - report what would happen, don't change anything.
# Skips the confirmation prompt automatically.
.\Remove-ShortcutVirus.ps1 -WhatIf

# Only clean USB drives, don't touch the host.
.\Remove-ShortcutVirus.ps1 -SkipUsb

# Only disinfect the host, don't touch USB drives.
.\Remove-ShortcutVirus.ps1 -SkipHost

# Skip the consolidated confirmation prompt (unattended runs / scripts).
.\Remove-ShortcutVirus.ps1 -Force

# Send the log somewhere specific.
.\Remove-ShortcutVirus.ps1 -LogPath C:\Logs\svr.log
```

| Flag        | Meaning                                                                 |
|-------------|-------------------------------------------------------------------------|
| `-WhatIf`   | Dry run. Logs every action without changing anything. No prompt.        |
| `-SkipUsb`  | Skip removable drive cleanup.                                           |
| `-SkipHost` | Skip host (System32 / services / Defender exclusions) cleanup.          |
| `-Harden`   | Disable AutoRun, show hidden files, enable Defender ASR rules.          |
| `-Force`    | Skip the consolidated confirmation prompt. Equivalent to auto-Yes.      |
| `-LogPath`  | Override the log path. Default lives under `%ProgramData%`.             |

### Recovery and standalone hardening

```powershell
# Recover from E: to Desktop\USB-Recovered-<timestamp>, with manifest.
.\Recover-UsbFiles.ps1 -Drive E:

# Apply prevention settings only (no disinfection).
.\Protect-System.ps1
```

`Recover-UsbFiles.ps1` writes a `recovery-manifest.json` at the
destination root with the SHA-256, size, and last-write-time of every
recovered file.

`Protect-System.ps1` is the same hardening that `-Harden` does in the
remover, packaged as a standalone script. It does not require an
infection to be present.
